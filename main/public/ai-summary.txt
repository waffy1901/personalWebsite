Website Structure and User-Facing Routes:

This portfolio is a Vite React application under the main/ directory. It uses
React Router and publishes the following primary routes:

* / - Home page with profile summary, Georgia Tech branding, social/contact
  links, resume download, deploy metadata, and AI-summary actions.
* /case-studies - selected engineering case studies for reliability,
  deployment recovery, and data reconciliation work.
* /case-studies/kubernetes-autoscaling - deeper writeup on deploying and
  validating Kubernetes Horizontal Pod Autoscaling for a transaction-critical
  production service.
* /case-studies/legacy-deployment-recovery - deeper writeup on reconstructing
  deployment workflows for legacy Java/Tomcat services and enabling
  zero-downtime Cassandra credential rotation.
* /case-studies/cdc-data-reconciliation - deeper writeup on the CDC data
  reconciliation platform, stakeholder feedback, and public health data
  comparison workflow.
* /projects - project cards for CDC Data Reconciliation, Job Search Aid, and
  Campus Discovery Service, each with tech stack, detail view, logo, and GitHub
  source link.
* /experience - work experience, extracurricular experience, and platform
  ownership cards.
* /resume - resume preview image with direct open and download actions for the
  PDF.
* /contact - Formspree-backed contact form, fallback email path, and LinkedIn,
  GitHub, and email links.
* /ai-summary.txt - this AI-readable portfolio summary.

Legacy uppercase routes such as /Resume, /Projects, /Experience, and /Contact
redirect to lowercase routes in the React app. Netlify redirects also preserve
single-page application routing and redirect /waffyahmedresume.pdf to the
canonical /waffyAhmedResume.pdf path.

  (2024-09-13T19:43:00Z) and a build-time last deployment date from
  VITE_DEPLOY_DATE, falling back to a local preview timestamp in development.

  changes without automatic duplicate page views.
* Contact form: uses VITE_FORMSPREE_KEY. If the key is absent, the app displays
  a graceful fallback with a direct mailto link.

Deployment, Security, and Repository Workflow:

* Netlify config builds from main/, runs npm run build, publishes main/dist,
  and uses Node 22.
* Production build injects VITE_DEPLOY_DATE and VITE_SITE_URL without rewriting
  local .env files.
* Security headers include a Content-Security-Policy, X-Content-Type-Options:
  nosniff, Referrer-Policy: strict-origin-when-cross-origin, and a restrictive
  Permissions-Policy.
* CSP allows self-hosted app assets, Google Tag Manager / Google Analytics
  scripts and connections, Formspree form submissions, and self-hosted frames
  for resume viewing.
* GitHub Actions includes a pull-request CI workflow for main that runs npm ci,
  lint, tests, and production build from main/.
* A main-branch deployment release workflow repeats verification and then
  creates a GitHub release tag in the format deploy-YYYYMMDDTHHMMSSZ-<short-sha>.
* Root package scripts forward dev, build, lint, test, and preview commands
  into main/.
